# Closure Policy

Apply this policy when producing final task outputs or closure records.

## 1. Every task must end with explicit closure
A complete closure should include:
- what was attempted
- what was changed or decided
- how it was verified
- what remains unresolved

## 2. Verification is mandatory
Verification must contain objective evidence where possible, such as:
- test results
- command output
- screenshot
- artifact path
- reproducible observation

## 3. Retro is mandatory
Capture at least one reusable lesson, rule, or improvement for future tasks.

## 4. No fake completeness
If the task is partial, say it is partial.
If validation is missing, say validation is missing.
If blockers remain, list blockers explicitly.

## 5. Optimize for future reuse
Closure text should help a later agent or human quickly understand:
- state
- confidence
- evidence
- next step
